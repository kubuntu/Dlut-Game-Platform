详细描述

封装发送游戏信息的底层通信

SubServer 需要向 MainServer 发送服务器信息、游戏信息及关联信息。

注解:
DGP API 提供了更加易于使用的 JSendGsInfo 类来处理发送游戏信息请求。

通信协议

首先，这个通信协议只定义了基本通信协议 中data的部分。
查看完整的通信协议，请见基本通信协议 。

  • 客户端向服务器端发送服务器信息 : <JID:protocol> <SubServer::SSubServer:subserverinfo>
    1. JID 类型，值为SubServerESP_ServerInfo，表示发送协议为发送服务器信息。
    2. SubServer::SSubServer ，表示服务器信息。格式化方式参见 operator<<(QDataStream& stream,const SubServer::SSubServer& )
  • 客户端向服务器端发送游戏信息 : <JID:protocol> <SubServer::SGameInfo2:gameinfo>
    1. JID 类型，值为SubServerESP_GameInfo，表示发送协议为发送游戏信息。
    2. SubServer::SGameInfo2 ，表示游戏信息。格式化方式参见 operator<<(QDataStream& stream,const SubServer::SGameInfo2& )
  • 客户端向服务器端发送服务器信息 : <JID:protocol> <JID:serverId> <JID:gameId> <JVersion:gameVersion>
    1. JID 类型，值为SubServerESP_Relation，表示发送协议为发送关联信息。
    2. JID 类型，表示服务器ID。
    3. JID 类型，表示游戏ID。
    4. JVersion 类型，表示游戏版本。格式化方式参见 operator<<(QDataStream& stream,const JVersion& )
  • 服务器端向客户端 ： <JID:protocol> <JCode:code>
    1. JID 类型，表示发送协议。
    2. JCode 类型，0表示发送成功。其它表示各种原因的失败。（有空把这个地方详细写一下。）
